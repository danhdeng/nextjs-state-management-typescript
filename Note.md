# use different tool/library to manage the state in nextjs app

# use-state : useState from React

https://reactjs.org/docs/hooks-state.html

# use-context: useContext from React

https://reactjs.org/docs/hooks-reference.html#usecontext

# use-react-query: ReactQuery data fetch library

https://react-query.tanstack.com/

https://github.com/tannerlinsley/react-query

# use-redux : React Redux library eact Redux is the official React binding for Redux. It allows React components to read data from a Redux Store, and dispatch Actions to the Store to update data.

https://react-redux.js.org/

# use-zustand : Zustand library is a small, fast and scalable bearbones state-management solution using simplified flux principles. Has a comfy api based on hooks, isn't boilerplatey or opinionated.

https://github.com/pmndrs/zustand

# use-mobx : Mobx library library that makes state management simple and scalable by transparently applying functional reactive programming (TFRP).

https://mobx.js.org/README.html

# use-Jotai: Jotai library is scalable from a simple React.useState replacement to a large-scale application with complex requirements.

https://jotai.org/

# use-rxjs : RxJS is a library for composing asynchronous and event-based programs by using observable sequences. It provides one core type, the Observable, satellite types (Observer, Schedulers, Subjects) and operators inspired by Array#extras (map, filter, reduce, every, etc) to allow handling asynchronous events as collections.

https://rxjs.dev/guide/overview
